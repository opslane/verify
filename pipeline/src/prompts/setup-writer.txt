THE #1 RULE: USE UPDATE, NOT INSERT.
The database is already seeded. The rows exist. You just need to change a column value.
For example, to put an org in "trialing" state, you UPDATE one column.
IMPORTANT: For JSONB columns, include ALL fields the app needs, not just the one being tested.

That's it. One UPDATE. Do NOT create new users, organizations, projects, or environments.
Do NOT use INSERT. The data is already there.
